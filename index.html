coordinates

Grids, point sets, projection and interpolation in Fortran

The coordinates library manages grids and arbitrary sets of points, and maps fields between them. It handles projection between coordinate systems, nearest-neighbor and conservative interpolation, and caches the (expensive) mapping calculation to disk so it is only ever done once per grid pair.

It is the workhorse behind regional/global climate and ice-sheet model pre-processing: taking observations or model output on one grid (a global lat-lon grid, an unstructured set of station locations, polygon data) and remapping them onto another (e.g. a regional oblique-stereographic grid over Greenland or Antarctica).

The mental model

Everything is built on points. A grid is just a structured 2D arrangement of points, and a map records, for each target point, which source points feed it and with what weight. Because the design pushes all the state into these three objects, mapping a field between two domains takes only three steps and no temporary bookkeeping variables.

The three-step example

use coord

type(grid_class) :: gCCSM3, gGRL
type(map_class)  :: map1

! 1. Define each grid (global lat-lon source, regional stereographic target)
call grid_init(gCCSM3, name="CCSM3-T42", mtype="latlon", units="degrees", &
               x0=0.d0,   dx=2.d0, nx=180, y0=-90.d0, dy=2.d0, ny=90)

call grid_init(gGRL, name="GRL-20KM", mtype="stereographic", units="kilometers", &
               dx=20.d0, nx=76, dy=20.d0, ny=151, &
               lambda=-40.d0, phi=72.d0, alpha=7.5d0)

! 2. Build (or load from cache) the mapping between them
call map_init(map1, gCCSM3, gGRL, max_neighbors=10)

! 3. Map a field
call map_field(map1, "t2m", var_ccsm3, var_grl, mask_map, method="quadrant")

The first map_init for a given grid pair computes neighbors and writes a NetCDF map file under maps/. Every later run loads that file instead of recomputing — so the up-front cost (often >1 minute on large grids) is paid once.

Installation in brief

The library depends only on a Fortran compiler and NetCDF (the bundled NCIO wraps it). The optional conservative-remapping-via-CDO path additionally calls the cdo command-line tool at runtime. Build the static library with make coordinates and link against libcoordinates/include/libcoordinates.a, then use coord in your program.
